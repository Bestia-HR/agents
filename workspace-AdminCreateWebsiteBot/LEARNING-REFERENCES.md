# Learning references (for AI agents)

Use these when users ask "where can I learn X?", "best resources for...", or when you want to cite authoritative, free learning sources. Prefer linking to official or highly-regarded community resources.

**In this workspace** you also have: `LEARNING-REFERENCES.md` (this file) and the same external links. For coding concepts use the project’s knowledge files in the main workspace; for “where to learn” point to the URLs below.

---

## Web fundamentals (HTML, CSS, JavaScript)

| Resource | URL | What it covers |
|----------|-----|----------------|
| **MDN Learn Web Development** | https://developer.mozilla.org/en-US/docs/Learn_web_development | Structured path: HTML, CSS, JS, accessibility, tooling. Interactive challenges, MDN Playground. |
| **MDN Web Docs** (reference) | https://developer.mozilla.org/en-US/docs/Web | Authoritative reference for HTML, CSS, JS, APIs. |
| **freeCodeCamp** | https://www.freecodecamp.org/learn | Project-based curriculum: Responsive Web Design, JS, React, Backend, etc. Free certifications. |
| **web.dev** (Google) | https://web.dev | Performance, accessibility, responsive design, modern CSS/JS. Guides and codelabs. |

## When to cite these

- User asks for learning resources or tutorials → suggest MDN Learn, freeCodeCamp, or the relevant section above.
- User asks about animation → GSAP docs, Three.js, or web.dev animations guide.
- User asks about accessibility → W3C WAI, MDN Accessibility, or web.dev a11y.
- User asks "best place to learn X" → pick the most relevant row and give the URL and a one-line description.

Keep answers concise; link to the reference rather than duplicating long tutorials.
